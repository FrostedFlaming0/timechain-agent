Slash commands

Type any of these in the message box (in the chat) or the terminal REPL. Each entry says what the command does and why you'd reach for it. Commands gate memory and integrity, not your replies.

Chain & integrity

Prove the record is intact and anchor checkpoints.

/verify

Cryptographically verify the whole chain — Ed25519 signatures, SHA-256 hash links, and Merkle batches.

Why: confirm no record was altered, reordered, or removed since it was sealed.

/verify-semantic

Schema-level consistency probe — dangling references, a revision pointing at a non-existent record, etc.

Why: catch the class of corruption the cryptography can't: the bytes are fine, but the meaning is inconsistent.

/length

Print the current record count.

Why: a quick read of how large the chain has grown.

/seal

Create a Merkle batch over recent un-batched records.

Why: anchor a compact, tamper-proof checkpoint over a span of history.

Memory & retrieval

Consolidate, correct, and pull back the relevant past.

/reflect

Have the agent summarize recent history into a reflection record.

Why: consolidate what mattered so salience-weighted retrieval surfaces it later.

/revise N <text>

Append a correction record targeting record N (never erases the original).

Why: fix a wrong or outdated memory while keeping the full audit trail.

/recall-index

Print the compact map of memory — one line per record with its labels and keywords.

Why: scan the whole chain at a glance and decide yourself what's relevant.

/recall-fetch <ids>

Pull the full content of the record ids you chose, budget-bounded.

Why: read exactly the blocks you picked from the index — no more, no less.

/recall <query>

A cheap pre-filter that delegates to the retriever and returns candidate briefs.

Why: narrow the field on a chain too large to scan, then fetch what you judge relevant.

Reasoning

Score candidates and reason through hard questions.

/poq <text>

Show the Proof-of-Quality verdict for a piece of text — brightness, grounding, assertiveness, and the SEAL/REVISE/FORCE_UNCERTAINTY/REJECT decision.

Why: see how the quality gate would judge a candidate before it's committed.

/think <query>

Fork faculty-lens perspectives of the agent, run an in-process tree search, and collapse to the highest-truth path (read-only — no subagents).

Why: structured, multi-perspective reasoning on a hard or high-stakes question.

Self-defense

Detect, freeze, and recover from a compromise.

/immune-status

Show the membrane status — whether sealing is locked, the safe height, and learned attack scars.

Why: see at a glance whether the chain is in lockdown and what attacks it has already learned.

/immune-scan [text]

Scan sealed memory (and an optional input) for a compromise — a covenant breach, tampering, or a known attack pattern.

Why: find the first compromised record if you suspect an injection slipped through.

/lockdown

Freeze sealing — only a recovery record may be appended until you roll back.

Why: stop the agent from writing anything while you investigate a suspected compromise.

/rollback <N>

Resume from the clean height before record N; quarantine the wound and learn its attack vector as a scar.

Why: recover from a compromise without erasing history — the wound stays as a learned scar.

Faculties & growth

Inspect and grow the agent's perceptual/reasoning faculties.

/modalities

List the built-in and sprouted modalities (status, domain flag, weight).

Why: see which detectors are active and how retrieval is anchored.

/cambium

Scan recent history for recurring gaps and commit proposals (it proposes; it never auto-applies).

Why: surface patterns that might deserve a new faculty or skill.

/cambium-full

A deep, one-shot scan of the entire chain (linear in length).

Why: catch long-gap recurring patterns the bounded incremental scan can miss.

/cambium-grow <text>

Grow a faculty when the input reveals a gap — fuse two close faculties or sprout a fresh one; promote it on recurrence.

Why: let the agent endogenously expand its faculties when it meets something its current ones can't cover.

/proposals

List proposal records, escalated ones first, with recurrence counts.

Why: review pending growth suggestions before deciding what to apply.

Long-horizon tasks

Work jobs larger than any context window, resumably.

/continuum-resume

Re-hydrate the latest long-horizon task's full state from its head block alone.

Why: pick up a big task exactly where you left off — a new session, hours or weeks later.

/continuum-validate

Check the task's progress invariants (monotonic progress, data-height band) plus chain integrity.

Why: confirm a long ingest stayed coherent end to end.

Files & sources

Bring in source material and keep it honest.

attach button / drag-drop

Ingest content — document, image, or pasted text. With an active task it lands in that task's chain and workspace; otherwise it becomes an identity-chain attachment (bytes stored content-addressed). The agent can do the same via its ingest_blob tool.

Why: give the agent durable, verifiable source material it can retrieve later.

/verify-source <idx>

Re-check an ingested file record against the live file on disk, git-aware (verified / source-mismatch / revision-drift / dirty-worktree …).

Why: catch acting on stale code or data that changed since it was ingested.

/migrate

Re-embed every record into the embedding index with the current embedder — a one-time, off-chain backfill (progress streams live).

Why: make an old chain — or one indexed with a different embedder — retrieve as richly as a fresh one. The signed records are never touched.

Consensus & sharing

Harden against forgery and exchange memory with peers.

/consensus-init [n] [k]

Create a k-of-n witness quorum that attests every chain head.

Why: add tamper-resistance on top of tamper-evidence — a forged record needs ≥k witness keys too.

/consensus-verify

Verify the chain and require k-of-n witness agreement on the recomputed head hash.

Why: confirm no record was forged even if it was re-signed.

/export-capsule <path>

Export shareable and public records as a signed, verifiable bundle (private and quarantined records never leave).

Why: share verifiable memory with another agent.

/import-capsule <path>

Verify and import another agent's capsule as attributed third-party memory.

Why: take in a peer's memory without silently treating it as your own ground truth.

System

Configuration and in-terminal help.

/sysprompt

Show the system-prompt history sealed on the chain.

Why: audit how the agent's configured behavior changed over time.

/cypher-help

List the faculty commands in the terminal REPL.

Why: a quick in-terminal reference without leaving the prompt.
